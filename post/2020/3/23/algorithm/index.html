排序算法

分类

排序算法可以分为内部排序和外部排序

内部排序是待排序记录在内存中进行排序

外部排序是因为排序的数据很大，一次不能容纳全部的排序记录，在排序过程中需要访问外存。

常见的内部排序算法

  • 交换排序（冒泡排序、快速排序）
  • 选择排序（简单选择排序、堆排序）
  • 插入排序（直接插入排序、希尔排序）
  • 归并排序、基数排序等。

冒泡排序

基本思想

将当前数据序列的各相邻数据两两比较，按照要求的升降顺序调换顺序。

假设待排序序列长度为n，则需要进行 n-1 次排序，第 i 次排序需要进行 n - i 次比较。（一般情况）

时间复杂度

最好（正序） 最坏（反序） 平均
O(n) O(n^2) O(n^2)

空间复杂度

O(1)

实现思路

1、比如待排序序列为 5,3,2,4,1 （从小到大） 长度n=5

2、第 1 次排序 3,2,4,1,5（每一次相邻元素进行比较） 共比较4( n-1 )次

3、第 2 次排序 2,3,1,4,5（每一次相邻元素进行比较） 共比较3( n-2 )次

4、第 3 次排序 2,1,3,4,5（每一次相邻元素进行比较） 共比较2( n-3 )次

5、第 4 次排序 1,2,3,4,5（每一次相邻元素进行比较） 共比较1( n-4 )次

6、每一次将最大的气泡（元素）浮出水面，浮出后就排好了。

     * @Description 冒泡排序 要求将数组按照从大到小排序 例：{1,4,6,3,7,9,2,8,1,23,3,4,5,6};
     * @author lossong
     * @date 2020年3月23日 下午20:47
     */
    public static void bubbleSort(int arr[]) {
        int temp = 0;
        // int recoder = 0; // @2
        // int len = arr.length - 1; // 优化2将内循环次数替换为len，原为 arr.length - i - 1
        for (int i = 0; i < arr.length - 1; i++) {
            // int flag = 1; //@1
            for (int j = 0; j < arr.length - i - 1; j++) {
                if (arr[j] < arr[j + 1]) {
                    temp = arr[j + 1];
                    arr[j + 1] = arr[j];
                    arr[j] = temp;
                    // flag = 0; //如果发生交换，标志位置0 @1
                    // recoder = j; // 记录交换为位置 @2
                }
                print(arr, i, j);
            }
            // len = recoder; //将交换的最后一次位置设置下次内循环的次数 @2
            print(arr, i);
            // if (flag == 1) return; // 标志位为1，则未交换，退出 @1
        }
    }

优化

1、当序列已经排好序了，就可以退出循环了，从而节省时间。（优化代码中@1处。）

优化思路：设置一个标志位，判断本次循环是否发生了交换，如果没有，则说明已经完成了排序，否则继续。

2、在内循环中，常常会有一些多于的比较，比如初始为6,5,7,4,3,1,2，第一次内循环结束后，序列为6,7,5,4,3,2,1，接下来的内循环中有的已经没有必要了。（优化代码中@2处）

优化思路：设置一个recorder，记录最后一次交换的位置，从而减少内循环次数。比如上面例子中最后一次

6,5,7,4,3,1,2 // 原始

6,7,5,4,3,2,1 // 最后一次为5，即下次内循环5次

7,6,5,4,3,2,1 // 最后一次为0，从而每次表示内循环的次数为0

快速排序

基本思想

通过找到一个基准，然后对前后数据进行遍历，找到该基准的正确索引。此时，该索引前的数据小于或等于它本身，其后的数据大于等于其本身。 然后通过递归调用，依次对前后的数据分别进行快排。直到全部排序完成。

时间复杂度

最坏为O(n^2),最好为O(nlog2*(n))

空间复杂度

最坏为O(n),最好为O(log2(n))

实现思路

1、设置两个指针初始分别指向区间的下界和上界。i指向下界，j指向上界。利用一个临时变量存储一个基准值。比如索引为0的值。 2、指针j自右向左扫描，直到扫描到第一个比基准小的值，则将R[j]处的值赋予R[i]处的值。(实现了基准值与R[j]的交换,值小于基准的到了基准的右边),如果扫描到的比基准大，则j–。 3、然后令i+1,开始自左向右的扫描（如果扫描到的比基准大，则j–）直到扫描到第一个比基准大的值，然后将R[i]处的值赋予R[j],然后重复第二步 4、直到i==j，该位置即为基准的位置 5、然后采用递归方式对前后两部分分别进行排序，直到排序完成。跳出递归的条件也就是low>=high。

/**
     * 
     * @Description 快排方法
     * @author lossong
     * @date 2019年8月27日 上午9:49:58 
     * @param arr
     * @param low
     * @param high
     */
    public static void quickSort(int[] arr, int low, int high) {
        //递归的中止条件low不小于high
        if(low<high) {
            //获取基准的最终位置
            int privot_index = getPrivot(arr, low, high);
            // 对左子序列进行排序
            quickSort(arr, low, privot_index - 1);
            // 对右子序列进行排序
            quickSort(arr, privot_index + 1, high);
        }
    }

    /**
     * 
     * @Description 获取基准值位置,进行交换
     * @author lossong
     * @date 2019年8月27日 上午9:50:14 
     * @param arr
     * @param low
     * @param high
     * @return 基准值最终索引位置
     */
    public static int getPrivot(int[] arr, int low, int high) {
        // 基准值
        int privot = arr[low];
        // 循环中止条件,当low==high时，对该部分排序完成
        while (low < high) {
            // 右边
            while (high > low && arr[high] >= privot) {
                high--;
            }
            arr[low] = arr[high];
            // 左边
            while (low < high && arr[low] <= privot) {
                low++;
            }
            arr[high] = arr[low];
        }
        arr[low] = privot;
        return low;
    }

优化

“三者取中”规则:在当前区间里，取该区间，区首、尾及中间位置的中间值作为基准，在开始排序前，先将它与区首值进行交换，然后再执行上面的操作。该优化针对已经有一定序列的排序。 “随机化”:即在low和high之间取一个随机值，使其作为基准。当然也和low位置的值进行交换，然后再排序。对于初始有序的序列，一般不会导致最坏的情况发生，能够提升算法的性能。
