数据库：按照某种数据结构对数据进行组织，存储和管理的容器。

如果没有数据库会存在的一些问题：

Size 数据量限制

Ease of updating 难以维护

Accuracy 同时操作

Security 数据安全

Redundancy 数据冗余

Backup 备份恢复

数据库管理软件(DBMS)：用于管理维护数据库的软件

关系数据库管理系统：基于关系模型的数据库管理系统称为关系数据库管理系统

比较流行的关系数据库管理系统：Oracle，MySQL，SQL Server，DB2等

关系数据库中所谓的关系，实质上是一张二维表

实体关系模型：该模型直接从现实世界中抽象出实体类型和实体间联系，然后用实体联系图（E-R图）表示数据模型，是描述概念世界，建立概念模型的实用工具

E-R图 矩形：实体 椭圆：属性 菱形：关系

实体间的关系: 一对一( 1 : 1 )，一对多( 1 : N )，多对多( M : N )

一对一的举例：比如学生与学号的关系

一对多的举例：比如班级与学生之间的关系

多对多的举例：比如说学生与课程间的关系

实体集->表

实体->行

实体属性->列

MySQL 关系型数据库

基本单元：表

主键: PK Primary Key（表中唯一，不重复）

主键是用来唯一标识表中的一条记录

主键可以由一个或多个字段（组合键）组成

主键不能为空

组合主键: Composite Key （当一个值不能唯一确定一个元素时，采用组合键作为主键）

外键: Fk Foreign Key（外键约束该属性的取值范围，只有主键才能做外键）

外键用于与另一张表的关联。是能确定另一张表记录的字段，用于保持数据的一致性

外键可以重复，可以为空

只有主键才能做外键

简单点说，外键就是约束了该列的取值范围

要体现1对1的关系，采用外键做主键，用在拆分一张表

要体现1对多的关系，一般采用主键做外键，外键放在多的那张表里。

要体现多对多的关系，一般采用连接表，将两张需要连接的表的主键作为外键存放进去

在设置外键时，有外键约束属性需要设置（四种）：

CASCADE：在父表进行删除或者更新时，子表同样会进行删除和更新匹配的记录。

SETNULL：在父表进行删除或者更新时，将子表中匹配的记录设置为null(因此子表的外键不能设置其not null)。

RESTRICT： 如果想要删除父表的记录，而在子表中有关联父表的记录，则不允许删除父表的记录。

NO ACTION：同RESTRICT一样。

三大范式

在设计表时要遵从三大范式（虽然不是绝对的）

第一范式:符合1NF的关系中的每个属性都不可再分。

第二范式:任何非关键字段应该依赖于整个主键

第三范式:没有非主键字段（除主键之外的列），依赖于另一个非主键字段，即没有冗余数据

有时为了提高运行效率，就必须降低范式标准，适当保留冗余数据。

具体做法是：在概念数据模型设计时遵守第三范式，降低范式标准的工作放到物理数据模型设计时考虑。

降低范式就是增加字段，允许冗余，达到以空间换时间的目的。比如在商品表中加入”金额”，尽管金额=数量*单价

范式化

优点：可以尽量减少数据冗余，数据更新更快

缺点：查询时需要对多个表进行关联，性能降低

反范式

优点：可以减少表的关联，查询的性能更好

缺点：存在数据冗余，且数据维护更难

结构化查询语言 SQL（Structured Query Language）

SQL是一种特殊目的的编程语言，是一种数据库查询和程序设计语言

用于存取数据以及查询、更新和管理关系数据库系统；同时也是数据库脚本文件的扩展名。

#导出
shell>mysqldump -u uname -p --databases db_name > 脚本文件地址 
#导入
mysql>source file_path
#如：
mysql>source E:/test.sql;

#查询

#查询表中所有信息
SELECT * FROM tb_name;
#查询某一列或多列信息
SELECT 列名1[,列名2,...] FROM tb_name;
#条件查询
SELECT 列名1[,列名2,...] FROM tb_name WHERE 条件;
条件中可以添加比较运算符号，逻辑运算符与或非（and, or, ！）
如：
SELECT stu_name 
FROM t_stu 
WHERE stu_age > 18 AND stu_age < 23;

SELECT a.stu_name,a.stu_birthday 
FROM t_student a            #a在这里的作用是给表取个别名
WHERE NOT (a.stu_id = '20127006' OR a.stu_id = '20127007');

在设计表时需要注意长度的设置，varchar(v_len) 和int(i_len) 其中，v_len代表的是定义的字段的长度，即可变长字符的最大长度；而数字类型的类型的范围限制仅与类型相关，int的大小是4字节，即范围为-2147483648~2147483647

而i_len并不代表可以存储的数值字符长度，它代表的是数据在显示时显示的最小长度，当存储的字符长度小于i_len值时，只有在设置了zerofill用0来填充，才能够看到效果，换句话就是说，没有zerofill，i_len值就是无用的。
