####作为一只程序员，需要了解的东西非常多，其中就有Git ####那什么是Git呢？ Git是目前世界上最先进的分布式版本控制系统 ####当我们的程序需要多个人编写时，团队之间的代码提交就需要这么一个管理平台，记录每次修改信息，保存了之前的版本。 ####所以呢接下来就学习下Git ####推荐教程 廖雪峰Git教程 安装好后。。。 ##windows下，在想作为仓库的目录下，右键选择Git Bash Here（前提是安装好了Git）就会出现linux下bash ###首先来初始化一个仓库 git init ###添加文件 git add <file> 如 git add ./src/index.html ###提交文件到当前分支 git commit -m "对提交文件的说明，比如修改内容等"

##上面的操作时创建自己的仓库，我们还可以去克隆别人的仓库。 ###1、克隆仓库到本地 git clone 仓库地址 和 git clone -b 分支名仓库地址 ###2、接下来可对文件修改，然后进行提交

1

git add file
1

git commit -m "description"

将本地的分支推送到远程仓库里 git push origin 分支名字 ##分支管理 ###创建分支 git branch newbranch ###切换分支 git checkout newbranch ###相当于创建并切换 git checkout -b newbranch 用git branch命令查看当前分支

*newbranch master

当我们在自己分支上完成自己的任务时，可以将分支合并到master分支上 git merge命令用于合并指定分支到当前分支,所以首先要切换到master分支

1

git checkout master
1

git merge newbranch

删除分支 git branch -d newbranch

##其他常用命令 显示仓库状态 git status 查看修改内容 git diff <file>,就是查看difference，显示的格式正是Unix通用的diff格式。 查看历史提交记录 git log 查看分支合并图 git log --graph 可查看提交版本号，同时减少输出信息 git log --pretty=oneline 在git中，版本号并不是1,2,3递增的数字，避免冲突？ 在Git中，用HEAD表示当前版本,上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100。 版本回退git reset --hard HEAD^,也可以叫版本号（部分即可） 但是版本号不一定好记，所以Git提供了一个命令git reflog用来记录你的每一次命令

工作区即之前创建的文件夹 版本库即隐藏目录.git
